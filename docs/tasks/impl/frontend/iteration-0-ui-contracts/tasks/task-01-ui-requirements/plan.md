# Задача 01 — Функциональные требования к экранам: план

## Цель

Создать документ `docs/spec/frontend-requirements.md`, фиксирующий функциональные требования ко всем 5 экранам, общим компонентам, аутентификации и дизайн-системе. Документ должен быть достаточен для того, чтобы разработчик мог приступить к реализации экрана, не обращаясь к другим источникам.

---

## Контекст

Источники для формирования требований:

- **`docs/vision.md`** — роли (student / teacher), сценарии, принцип тонкого клиента
- **`docs/data-model.md`** — доменные сущности и их поля (что можно отобразить)
- **`docs/tasks/tasklist-frontend.md`** — состав экранов и ожидаемые компоненты

Требования должны быть согласованы с уже принятыми решениями: JWT httpOnly cookie, роли из `user_role` enum (`student` / `teacher`), shadcn/ui + Tailwind CSS.

---

## Что будет создано

| Файл | Действие |
|------|----------|
| `docs/spec/frontend-requirements.md` | Создать |

Структура документа:

1. **Экран 1 — Форма входа** (`/login`)
   - Цель, доступные роли
   - Компоненты: поля email + пароль, кнопка «Войти», `Alert` для ошибок
   - Действия: отправка формы → `POST /v1/auth/login` → редирект по роли
   - Edge cases: неверные данные, пустые поля, состояние загрузки

2. **Экран 2 — Календарь Преподавателя** (`/teacher/calendar`)
   - Цель, доступные роли: только teacher
   - Компоненты: недельная сетка 7 дней, `LessonCard` с 5 флагами, навигация по неделям
   - Боковые панели: лента bot-запросов (10 шт.), неподтверждённые занятия (2 дня), несданные ДЗ (2 дня), запросы на перенос
   - Действия: CRUD занятий через диалог, переключение флагов, кнопки «Напомнить»
   - Edge cases: пустая неделя, нет bot-запросов, нет неподтверждённых

3. **Экран 3 — Ученики** (`/teacher/students`, `/teacher/students/{id}`)
   - Цель, доступные роли: только teacher
   - Компоненты: переключатель карточки/таблица, `StudentCard`, `StudentTable`
   - Детальная форма: профиль, статистика (счётчики, прогресс-бар ДЗ), история занятий, лента диалога с ботом
   - Действия: CRUD учеников, бесконечный скролл диалога
   - Edge cases: ученик без telegram_id, пустая история

4. **Экран 4 — Настройки системы** (`/teacher/settings`)
   - Цель, доступные роли: только teacher
   - Компоненты: форма с 4 полями (имя репетитора, длительность по умолчанию, интервал напоминания о занятии, интервал напоминания о ДЗ)
   - Действия: загрузка текущих значений, сохранение, сброс к сохранённым
   - Edge cases: ошибка сохранения, невалидные значения (числа вне диапазона)

5. **Экран 5 — Расписание Ученика** (`/student/schedule`)
   - Цель, доступные роли: только student
   - Компоненты: переключатель месяц/неделя, месячная сетка с подсветкой дней, недельный вид с карточками, попап деталей занятия
   - Действия: навигация по месяцам/неделям, клик по занятию → детали
   - Edge cases: нет занятий в периоде

6. **Общие компоненты**
   - Боковая навигация: пункты по роли, активный пункт через `usePathname`
   - Header: название страницы, `ThemeToggle`, аватар с fallback, кнопка «Выйти»
   - Мобильный drawer: `Sheet` с `SheetTitle` (accessibility)
   - Адаптивность: sidebar скрыт на mobile, hamburger показывает drawer

7. **Аутентификация**
   - Механизм: JWT в httpOnly cookie (нет доступа из JS)
   - Роли: `teacher` → `/teacher/calendar`, `student` → `/student/schedule`
   - Middleware: проверка cookie → редирект на `/login` без токена; кросс-ролевой доступ блокируется
   - Без email-верификации в MVP

8. **Дизайн-система**
   - Библиотека компонентов: shadcn/ui
   - CSS-фреймворк: Tailwind CSS v4
   - Темы: light / dark, переключение пользователем, сохранение в localStorage, без FOUC
   - Адаптивность: все формы и экраны работают на mobile (375px+) и desktop (1280px+)
   - Иконки: из `iconLibrary` shadcn-конфига (определяется при инициализации проекта)

---

## Ключевые решения, которые нужно зафиксировать

| Решение | Обоснование |
|---------|-------------|
| Аутентификация без email-верификации | MVP: один преподаватель, ученики добавляются вручную |
| JWT httpOnly cookie (не localStorage) | Безопасность: недоступен из JS |
| shadcn/ui + Tailwind CSS | Указано в tasklist-frontend.md и vision.md |
| 5 bool-флагов в занятии | notification_sent, confirmed_by_student, homework_sent, solution_received, solution_checked |
| Бесконечный скролл диалога (не пагинация) | UX: лента сообщений естественно скроллится вверх |
