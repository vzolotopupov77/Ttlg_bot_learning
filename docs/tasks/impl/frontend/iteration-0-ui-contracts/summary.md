## Ключевые решения (зафиксировано в документах)

- **Логин = e-mail:** поле `email` в `POST /v1/auth/login`; в UI допускается подпись «Логин» или «Email», валидация формата e-mail.
- **Флаги занятия:** сохраняются сразу при переключении (`PATCH /v1/lessons/{id}/flags`).
- **Расписание ученика:** только **недельный** вид в требованиях; `GET /v1/student/schedule?week_start=`.
- **Заметки ученика:** поле `notes` в API/форме; добавление в БД — итерация 1 (`users.notes` или эквивалент).
- **Настройки:** в референсе одна кнопка «Сохранить»; «Сбросить» — опционально при реализации.
