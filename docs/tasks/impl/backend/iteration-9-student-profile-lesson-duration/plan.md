# Итерация 9: профиль ученика и длительность занятия

## Цель

Расширить модель `User` (класс, телефон, email) и `Lesson` (`duration_minutes`, по умолчанию 60), согласовать API и документацию.

Корректное хранение контактов и класса ученика; явная длительность занятия для расписания и напоминаний.

- ORM, Alembic-миграция, репозитории, FastAPI-схемы, smoke-тесты
