# Задача 20: устойчивость и логи

**Статус:** завершена (см. [summary.md](summary.md)).

## Что меняется

- Аудит HTTP-клиентов: `LLMClient` (backend), `BackendClient` (бот).
- При необходимости — явные `httpx.Timeout` (connect/read) для бота.
- Проверка логов: без ключей API, без заголовка `Authorization` в явном виде.
- Тесты при обнаружении зазоров (цепочка 503 уже покрыта интеграционно).
- Обновление `docs/vision.md` — только если появляются новые явные правила.

## Definition of Done

- Сбой LLM не роняет процесс; пользователь видит краткий fallback (503 → текст в Telegram).
- В логах нет полного Authorization header / сырого ключа.
