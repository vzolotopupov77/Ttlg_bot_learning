# Задача 05: Конвенции API — summary

## Сделано

- Добавлен [docs/api-conventions.md](../../../../../api-conventions.md): префикс `/v1`, единое тело ошибки (`error.code`, `error.message`), таблица HTTP-статусов, заглушка `X-User-Role`.
- [docs/integrations.md](../../../../../integrations.md) ссылается на конвенции из раздела Backend HTTP API.
- В `api-conventions.md` добавлен подраздел **«Нормализация `422` в реализации»**: FastAPI/Pydantic по умолчанию отдаёт `{"detail": [...]}` — зафиксирован обработчик `RequestValidationError`, который нормализует тело к единому формату `{"error": {...}}`.

## Отклонения / решения

- Нормализация `422` добавлена по итогам проверки `api-design-principles`: без неё клиент обрабатывал бы два разных формата ошибки.

## Проверка

- Пример ошибки, таблица кодов и обработчик `422` присутствуют в `api-conventions.md`.
