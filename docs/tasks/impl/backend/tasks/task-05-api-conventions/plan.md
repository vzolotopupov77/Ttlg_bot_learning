# Задача 05: Конвенции API (префикс, ошибки, коды)

## Цель

Зафиксировать единый префикс версии (`/v1`), формат тела ошибки и типовые HTTP-коды для клиентов (бот, будущий веб).

## Что меняется

- Новый файл [docs/api-conventions.md](../../../../../api-conventions.md).
- [docs/integrations.md](../../../../../integrations.md): ссылка на конвенции из раздела Backend HTTP API.

## Содержание конвенций (источник правды)

1. **Префикс:** все публичные маршруты API — под `/v1` (например `/v1/dialogue/message`, будущий `/v1/users/...`).
2. **Формат ошибки:** обёртка `error` с полями `code` (строковый машинный код) и `message` (краткий текст; без секретов и PII).
3. **Таблица кодов:** `200`, `201`, `400`, `404`, `422`, `500`, `503` — когда применять; согласование с FastAPI (`422` для Pydantic).
4. **Роли (заглушка):** до JWT/session — опциональный заголовок `X-User-Role`: `student` | `teacher` для будущих проверок; на MVP диалога может не проверяться.

## Definition of Done

- В `api-conventions.md` есть пример JSON ошибки и таблица HTTP.
- Из `integrations.md` есть явная ссылка на `api-conventions.md`.
